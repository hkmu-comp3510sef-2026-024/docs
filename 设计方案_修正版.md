# **图书借阅与会员管理系统 \- 设计方案**

## **项目概述**

本系统是一套完整的图书借阅与会员管理解决方案，采用前后端分离架构，包含会员端、管理端和后端API三个独立子项目。系统实现了从会员注册、图书检索、借阅预约、到管理端审核、借还办理、罚款管理等完整的业务流程。

 

### **技术架构**

**后端技术栈**

* Web框架：Express.js 5.2.1

* ORM框架：Prisma 7.4.2

* 数据库：MySQL/MariaDB

* 认证方案：JWT (jsonwebtoken 9.0.3) \+ bcryptjs 3.0.3

* 定时任务：node-cron 4.2.1

* 数据验证：Joi 18.0.2

* 文件上传：Multer 2.1.0

 

**前端技术栈（会员端 & 管理端）**

* 框架：Vue 3.5.25 \+ Composition API

* 构建工具：Vite 7.3.1

* UI组件库：Element Plus 2.13.2

* 状态管理：Pinia 3.0.4

* 路由：Vue Router 5.0.3

* HTTP客户端：Axios 1.13.6

 

### **数据模型概览**

系统包含12个核心数据表：

* **User** \- 用户表（会员、管理员、馆员）

* **Book** \- 书目表

* **Copy** \- 副本表

* **Loan** \- 借阅记录表

* **Reservation** \- 预约记录表

* **Fine** \- 罚款表

* **Notification** \- 通知表

* **AuditLog** \- 审计日志表

* **ReminderPolicy** \- 提醒策略表

* **AssistantSession \- 问答会话表**

* **AssistantMessage \- 问答消息表**

* **AssistantKnowledge \- 问答知识库表**

 

### **角色权限设计**

* **MEMBER \- 会员：可浏览图书、发起借阅、管理个人借阅和预约、查看通知、使用智能问答助手**

* **LIBRARIAN \- 馆员：可管理书目和副本、办理借还、处理罚款**

* **ADMIN \- 管理员：拥有馆员全部权限，并可管理会员、查看审计日志**

### **8\. 图书推荐模块（Recommendations）【创新点 A】**

**功能概述**

 

基于可解释规则引擎为会员生成个性化推荐，每次返回 5 本推荐图书并附带推荐理由，推荐结果过滤已借阅书籍并按 bookId 去重。推荐来源包括同分类热门、同作者推荐、新上架图书三类；新用户（无借阅历史）以全馆热门 \+ 新书组合作为冷启动方案。

 

**前端UI块**

 

**推荐图书卡片流（RecommendationList）**  
UI元素：推荐理由透明化标签（橙色标签，如"你常借的分类""同一作者""本月新书"）、封面缩略图、书名、作者、可借副本数、"立即借阅" / "加入预约"按钮；换一批按钮（刷新推荐结果）。  
交互：点击图书卡片跳转图书详情页；点击"换一批"重新请求推荐接口刷新列表；推荐标签鼠标悬停时展示推荐理由详情。

 

**推荐算法逻辑（后端实现）**

1\. **同分类热门**：统计会员最近借阅记录所涉及的分类，推荐该分类下借阅次数最多的书目（排除已借）

2\. **同作者推荐**：统计会员借阅历史中出现频次最高的作者，推荐该作者的其他馆藏作品（排除已借）

3\. **新上架推荐**：推荐近 30 天内上架的新书

4\. **冷启动方案**（无借阅历史）：推荐全馆热门书目与新上架书目的组合

5\. 所有结果按 bookId 去重，确保推荐5本不重复的图书

 

**后端接口**

* 获取个性化推荐列表：GET /api/member/recommendations  
  返回：\[{ bookId, title, author, reason, reasonType }\]（共 5 条）

 

 

### **9\. 智能问答助手模块（AI Assistant）【创新点 C】**

**功能概述**

为提升系统易用性与用户咨询效率，系统新增智能问答助手模块。该模块面向用户提供基于知识库的通用问答服务，支持对系统使用方法、借阅规则、预约规则、续借规则、罚款规则等常见问题进行自然语言问答。

本模块仅提供信息查询与规则说明，不涉及会员个人借阅数据、预约数据、罚款数据等实时业务信息，也不直接执行借阅、续借、取消预约等业务操作；当用户问题超出系统知识库范围时，统一提示联系馆员或管理员。

**前端UI块**

问答入口组件UI元素：导航栏“智能助手”入口、首页快捷提问按钮。交互：用户点击入口后打开问答面板，可在任意页面发起提问；支持未登录状态下使用。

会话面板UI元素：欢迎语、快捷问题标签（如“借书规则”“预约说明”“罚款规则”“如何注册”）、消息展示区、问题输入框、发送按钮、清空会话按钮。交互：用户输入问题后系统返回对应答案；点击快捷问题可直接发送；点击清空会话后清空当前问答内容。

答案展示卡片UI元素：答案正文、问题分类标签、提示说明文字。交互：当问题命中知识库时展示对应答案；当问题未命中时展示统一兜底提示，如“未找到相关答案，请联系馆员或管理员”。

**问答逻辑设计（后端实现）**

1\. 对用户输入问题进行基础分类，如借阅规则类、预约规则类、续借规则类、罚款规则类、系统使用类；

2\. 按问题分类从知识库中检索对应标准答案；

3\. 若存在匹配结果，则返回对应答案内容；

4\. 若无匹配结果，则返回统一兜底提示，引导用户联系馆员或管理员；

**后端接口**

创建问答会话：POST /api/assistant/sessions 返回：{ sessionId }

发送问题：POST /api/assistant/chat 请求参数：{ sessionId, question } 返回：{ answer, category }

获取会话历史：GET /api/assistant/sessions/{sessionId}/messages

清空当前会话：DELETE /api/assistant/sessions/{sessionId}

获取快捷问题：GET /api/assistant/quick-questions

## **三、管理端（Admin / Librarian Portal）**

### **10\. 会员审核管理模块（Member Management）**

**功能概述**

 

管理员可查看全体会员列表（支持按状态筛选），并对不同状态的会员执行相应操作：PENDING → 审核通过（设置有效期，状态变为 ACTIVE）；ACTIVE/EXPIRED → 冻结（填写冻结原因，状态变为 FROZEN）；FROZEN → 解冻（恢复为 ACTIVE）；任意状态 → 续期（延长有效截止日期）。以上所有操作均自动写入审计日志。

 

**前端UI块**

 

**会员列表（MemberList）**  
UI元素：状态筛选 Tab（全部 / PENDING / ACTIVE / EXPIRED / FROZEN，待审核 Tab 显示数量徽章）、关键字搜索框（按姓名/邮箱/学号搜索）、列表表格（姓名、邮箱、学号/借阅证号、状态标签、有效期、注册时间、操作按钮组）、分页器。  
交互：点击操作按钮弹出对应操作弹窗（审核通过/冻结/解冻/续期）；操作成功后刷新列表并显示 Toast 提示。

 

**操作弹窗组（MemberActionModals）**  
UI元素：审核通过弹窗（会员有效期日期选择器 \+ 确认/取消按钮）；冻结弹窗（冻结原因文本输入框 \+ 确认/取消按钮）；解冻弹窗（确认弹窗）；续期弹窗（新有效截止日期选择器 \+ 确认/取消按钮）。

 

**后端接口**

* 获取会员列表：GET /api/admin/members  
  参数：{ status, keyword, page, pageSize }

* 审核通过：POST /api/admin/members/{memberId}/approve  
  请求参数：{ endDate }

* 冻结会员：POST /api/admin/members/{memberId}/freeze  
  请求参数：{ reason }

* 解冻会员：POST /api/admin/members/{memberId}/unfreeze

* 续期：POST /api/admin/members/{memberId}/renew  
  请求参数：{ newEndDate }

 

 

### **11\. 书目管理模块（Book Management）**

**功能概述**

 

馆员或管理员可新增书目（填写 ISBN、书名、作者、分类、简介、出版年份、封面图片）、编辑已有书目基本信息，以及执行下架操作（is\_active 置为 false，下架后不再出现在会员端检索结果中）。所有操作自动写入审计日志。

 

**前端UI块**

 

**书目列表（BookManageList）**  
UI元素：关键字搜索框、分类筛选、上下架状态筛选（全部 / 上架 / 下架）、新增书目按钮（主要按钮）、列表表格（封面缩略图、书名、作者、ISBN、分类、副本总数、上架状态、操作按钮：编辑 / 下架 / 查看副本）、分页器。  
交互：点击新增/编辑按钮打开书目编辑弹窗；点击下架弹出二次确认弹窗，确认后执行下架并刷新列表；点击"查看副本"跳转至该书目的副本管理页。

 

**书目编辑弹窗（BookEditModal）**  
UI元素：封面图上传区、ISBN 输入框、书名输入框、作者输入框、分类下拉选择器、出版年份输入框、内容简介多行文本框、保存 / 取消按钮。

 

**后端接口**

* 获取书目列表：GET /api/admin/books  
  参数：{ keyword, category, isActive, page, pageSize }

* 新增书目：POST /api/admin/book  
  请求参数：{ isbn, title, author, category, description, publishYear, coverUrl }

* 编辑书目：PUT /api/admin/book/{bookId}

* 下架书目：PUT /api/admin/book/{bookId}/deactivate

**后端接口**

### **13\. 借还办理模块（Circulation）**

**功能概述**

 

核心业务模块，包含借出办理与归还办理。借出时：馆员录入会员信息与图书条码，系统创建借阅记录、将副本状态变为 ON\_LOAN 并设置应还日期；若该会员对该书有 READY\_FOR\_PICKUP 预约，系统自动将其标记为 COMPLETED 并与借阅关联。归还时：馆员记录图书状况（正常/损坏/丢失），系统更新借阅状态为 RETURNED，副本状态依状况变为 AVAILABLE / MAINTENANCE / LOST，自动计算逾期并生成罚款记录；归还后系统自动推进预约队列（将队首 QUEUED 者提升为 READY\_FOR\_PICKUP 并通知）。所有操作写入审计日志。

 

**前端UI块**

 

**借出办理面板（CheckoutPanel）**  
UI元素：会员搜索框（输入姓名/学号/邮箱，下方展示会员信息卡：姓名、状态、当前在借数量）、图书条码输入框（支持扫码枪输入）、图书信息预览区（书名、副本状态确认）、应还日期展示（系统自动计算）、确认借出按钮。  
交互：会员状态非 ACTIVE 时给出警告提示并禁止借出；副本状态非 AVAILABLE 时提示不可借；确认借出成功后显示借阅记录摘要。

 

**归还办理面板（ReturnPanel）**  
UI元素：图书条码输入框（支持扫码枪输入）、借阅记录信息预览区（会员、书名、借阅日期、应还日期、是否逾期及逾期天数）、图书状况单选按钮（正常 / 损坏 / 丢失）、确认归还按钮；逾期时显示预计罚款金额提示（橙色）。  
交互：条码识别后自动拉取借阅记录并计算逾期情况；选择"损坏"或"丢失"状况时弹出二次确认；确认归还成功后显示归还结果摘要（含罚款金额及预约推进情况）。

 

**后端接口**

* 借出办理：POST /api/circulation/checkout  
  请求参数：{ memberId, copyBarcode }

* 归还办理：POST /api/circulation/return  
  请求参数：{ copyBarcode, condition: normal|damaged|lost }

* 按条码查询借阅记录（归还预校验）：GET /api/circulation/lookup?barcode={barcode}

 

 

### **14\. 罚款管理模块（Fine Management）**

**功能概述**

 

馆员或管理员可查看罚款列表（支持按状态筛选：UNPAID / PAID / WAIVED），对未缴罚款执行"标记已缴"（Mock 支付）或"减免"操作，两种操作均记录操作人与时间。罚款由系统在归还时自动计算生成（逾期天数 × 每日罚款额，不超过最大上限，支持设置宽限期）；所有状态变更写入审计日志。

 

**前端UI块**

* 减免罚款：POST /api/admin/fines/{fineId}/waive

## **四、系统后台任务模块（Scheduler / Jobs）**

### **17\. 到期提醒任务（Due Reminder Job）**

**执行周期**：每天 08:00

 

**功能概述**  
每天在固定时间自动运行，扫描全部"借阅中且未归还"的记录，结合当前提醒策略判断是否需在当日发送到期提醒通知。内置防重复机制：同一借阅记录在同一日期同一类型下仅生成一条通知，避免重复打扰用户。

 

**执行逻辑**

1\. 查询所有状态为"借阅中"的记录；

2\. 计算今日距应还日期的天数差，判断是否命中策略中配置的到期前提醒节点（如 3、1、0）；

3\. 命中节点且该借阅记录今日该类型通知尚未生成，则创建一条 DUE\_REMINDER 类型站内通知并关联至对应会员。

 

### **18\. 逾期提醒任务（Overdue Reminder Job）**

**执行周期**：每天 08:30

 

**功能概述**  
每天自动运行，扫描所有已超过应还日期但尚未归还的借阅记录，依据当前逾期提醒策略（如逾期第 1、3、7 天）在对应天数生成催还通知。同样采用防重复机制确保同一借阅在同一逾期天数节点下仅生成一条通知。

 

**执行逻辑**

1\. 查询所有应还日期早于今日且状态为"借阅中"的记录；

2\. 计算当前逾期天数，判断是否命中策略中配置的逾期后提醒节点（如 1、3、7）

3\. 命中节点且该借阅记录在该逾期天数下的 OVERDUE\_REMINDER 通知尚未生成，则创建通知并关联至对应会员。

 

### **19\. 预约超时顺延任务（Reservation Expire and Shift Job）**

**执行周期**：每 30 分钟

 

**功能概述**  
每隔 30 分钟自动运行，扫描处于 READY\_FOR\_PICKUP 状态且取书截止时间已过的预约记录，将其状态标记为 EXPIRED 并通知用户，随后自动提升该书目队列中下一位 QUEUED 用户为 READY\_FOR\_PICKUP 状态，重新设置取书截止时间并发送到书通知。该机制确保预约队列持续自动流转，不因某位用户未及时取书而阻塞后续排队者。

 

**执行逻辑**

1\. 查询所有状态为 READY\_FOR\_PICKUP 且 pickupDeadline \< 当前时间的预约记录；

2\. 将上述记录状态更新为 EXPIRED，生成 RESERVATION\_EXPIRED 类型站内通知推送至对应会员；

3\. 对每个涉及的书目，在其预约队列中查找下一位 QUEUED 状态的预约记录，将其状态更新为 READY\_FOR\_PICKUP，设置新的取书截止时间，并生成 RESERVATION\_READY 类型通知推送至该会员。若队列已无等待者则跳过此步。

 

 

## **五、业务流程说明**

### **借阅流程**

1\. 会员登录系统，在图书检索页面搜索图书

2\. 进入图书详情页，查看馆藏信息（可借副本数、在借数量、维修中数量、预约队列长度）

3\. 如有可借副本，点击"立即借阅"按钮发起借阅

4\. 系统自动分配或会员手动指定可用副本

5\. 馆员在管理端借还办理模块扫描条码完成借出

6\. 系统创建借阅记录，设置应还日期，更新副本状态为 ON\_LOAN

 

### **预约流程**

1\. 会员在图书详情页发现无可借副本

2\. 点击"加入预约队列"按钮创建预约记录

3\. 预约状态为 QUEUED，显示排队位置

4\. 当有副本归还后，系统自动将队首预约提升为 READY\_FOR\_PICKUP

5\. 系统发送预约到书通知，设置取书截止时间

6\. 会员在截止日期前到馆办理借阅，预约标记为 COMPLETED

 

### **归还流程**

1\. 馆员在管理端借还办理模块扫描图书条码

2\. 系统拉取借阅记录，计算逾期天数和罚款金额

3\. 馆员选择图书状况（正常/损坏/丢失）

4\. 确认归还后，系统更新借阅状态为 RETURNED

5\. 根据图书状况更新副本状态（AVAILABLE/MAINTENANCE/LOST）

6\. 如有逾期，自动生成罚款记录

7\. 系统自动推进该书的预约队列

 

### **逾期罚款计算**

* 宽限期（graceDays）内：不计罚款

* 超过宽限期后：罚款金额 \= (逾期天数 \- 宽限天数) × 每日罚款额

* 罚款上限：不超过策略中设置的最高罚款上限（maxFineAmount）

 

### **预约队列顺延机制**

* 当 READY\_FOR\_PICKUP 状态的预约超过取书截止时间时

* 系统自动将其标记为 EXPIRED 并通知用户

* 同时自动将该书的下一个 QUEUED 预约提升为 READY\_FOR\_PICKUP

* 新的取书截止时间 \= 当前时间 \+ 预约保留期（系统配置）

 

 

## **六、数据状态机设计**

### **会员状态转换**

           
    PENDING (待审核)      
        └─ 审核通过 ──\> ACTIVE (有效)      
           
    ACTIVE (有效)      
        ├─ 过期 ──\> EXPIRED (已过期)      
        └─ 冻结 ──\> FROZEN (已冻结)      
           
    EXPIRED (已过期)      
        ├─ 续期 ──\> ACTIVE (有效)      
        └─ 冻结 ──\> FROZEN (已冻结)      
           
    FROZEN (已冻结)      
        └─ 解冻 ──\> ACTIVE (有效)      
         

 

### **借阅状态转换**

           
    ACTIVE (借阅中)      
        ├─ 归还 ──\> RETURNED (已归还)      
        └─ 续借 ──\> ACTIVE (借阅中，续借次数+1，应还日期延长)      
         

 

### **预约状态转换**

           
    QUEUED (排队中)      
        ├─ 到书 ──\> READY\_FOR\_PICKUP (可取书)      
        └─ 取消 ──\> CANCELLED (已取消)      
           
    READY\_FOR\_PICKUP (可取书)      
        ├─ 借阅 ──\> COMPLETED (已完成)      
        ├─ 超时 ──\> EXPIRED (已过期)      
        └─ 取消 ──\> CANCELLED (已取消)      
         

 

### **副本状态转换**

           
    AVAILABLE (在馆可借)      
        ├─ 借出 ──\> ON\_LOAN (外借中)      
        └─ 维修 ──\> MAINTENANCE (维修中)      
           
    ON\_LOAN (外借中)      
        ├─ 归还(正常) ──\> AVAILABLE (在馆可借)      
        ├─ 归还(损坏) ──\> MAINTENANCE (维修中)      
        └─ 归还(丢失) ──\> LOST (丢失)      
           
    MAINTENANCE (维修中)      
        └─ 维修完成 ──\> AVAILABLE (在馆可借)      
           
    LOST (丢失)      
        └─ 找回 ──\> AVAILABLE (在馆可借)      
           
    任意状态 ──\> REMOVED (下架)      
         

 

### **罚款状态转换**

           
    UNPAID (未缴)      
        ├─ 缴费 ──\> PAID (已缴)      
        └─ 减免 ──\> WAIVED (已减免)

## **九、创新点总结**

### **创新点 A：可解释的图书推荐系统**

基于规则引擎的透明化推荐系统，为用户提供可理解的推荐理由：

* 同分类热门推荐：根据用户借阅历史推荐热门分类图书

* 同作者作品推荐：推荐用户常借作者的其他作品

* 新书推荐：推荐最新上架的图书

* 推荐理由透明化：用户可清晰了解推荐原因

 

### **创新点 B：灵活可配置的提醒策略系统**

提供高度可配置的提醒和罚款策略，满足不同图书馆的管理需求：

* 到期前提醒天数可配置：支持设置多个提醒时间点

* 罚款策略可配置：每日罚款金额、最高罚款上限、宽限天数

* 策略修改即时生效：无需重启系统

* 操作可追溯：策略变更记录完整写入审计日志

 

### **创新点 C：智能问答助手**

基于知识库提供通用问答服务，支持用户通过自然语言快速获取系统使用说明与借阅规则说明，降低系统使用门槛，提升咨询效率；同时可通过持续补充知识库，不断完善常见问题覆盖范围。

核心优势

* 基于知识库提供通用问答服务

* 支持自然语言获取系统说明和借阅规则说明

* 降低使用门槛

* 可通过持续补充知识库完善常见问题覆盖范围

3\. 如有可借副本，点击"立即借阅"按钮发起借阅

4\. 系统自动分配或会员手动指定可用副本

5\. 馆员在管理端借还办理模块扫描条码完成借出

6\. 系统创建借阅记录，设置应还日期，更新副本状态为 ON\_LOAN

 

### **预约流程**

1\. 会员在图书详情页发现无可借副本

2\. 点击"加入预约队列"按钮创建预约记录

3\. 预约状态为 QUEUED，显示排队位置

4\. 当有副本归还后，系统自动将队首预约提升为 READY\_FOR\_PICKUP

5\. 系统发送预约到书通知，设置取书截止时间

6\. 会员在截止日期前到馆办理借阅，预约标记为 COMPLETED

 

### **归还流程**

1\. 馆员在管理端借还办理模块扫描图书条码

2\. 系统拉取借阅记录，计算逾期天数和罚款金额

3\. 馆员选择图书状况（正常/损坏/丢失）

4\. 确认归还后，系统更新借阅状态为 RETURNED

5\. 根据图书状况更新副本状态（AVAILABLE/MAINTENANCE/LOST）

6\. 如有逾期，自动生成罚款记录

7\. 系统自动推进该书的预约队列

 

### **逾期罚款计算**

* 宽限期（graceDays）内：不计罚款

* 超过宽限期后：罚款金额 \= (逾期天数 \- 宽限天数) × 每日罚款额

* 罚款上限：不超过策略中设置的最高罚款上限（maxFineAmount）

 

### **预约队列顺延机制**

* 当 READY\_FOR\_PICKUP 状态的预约超过取书截止时间时

* 系统自动将其标记为 EXPIRED 并通知用户

* 同时自动将该书的下一个 QUEUED 预约提升为 READY\_FOR\_PICKUP

* 新的取书截止时间 \= 当前时间 \+ 预约保留期（系统配置）

 

 

## **六、数据状态机设计**

### **会员状态转换**

           
    PENDING (待审核)      
        └─ 审核通过 ──\> ACTIVE (有效)      
           
    ACTIVE (有效)      
        ├─ 过期 ──\> EXPIRED (已过期)      
        └─ 冻结 ──\> FROZEN (已冻结)      
           
    EXPIRED (已过期)      
        ├─ 续期 ──\> ACTIVE (有效)      
        └─ 冻结 ──\> FROZEN (已冻结)      
           
    FROZEN (已冻结)      
        └─ 解冻 ──\> ACTIVE (有效)      
         

 

### **借阅状态转换**

           
    ACTIVE (借阅中)      
        ├─ 归还 ──\> RETURNED (已归还)      
        └─ 续借 ──\> ACTIVE (借阅中，续借次数+1，应还日期延长)      
         

 

### **预约状态转换**

           
    QUEUED (排队中)      
        ├─ 到书 ──\> READY\_FOR\_PICKUP (可取书)      
        └─ 取消 ──\> CANCELLED (已取消)      
           
    READY\_FOR\_PICKUP (可取书)      
        ├─ 借阅 ──\> COMPLETED (已完成)      
        ├─ 超时 ──\> EXPIRED (已过期)      
        └─ 取消 ──\> CANCELLED (已取消)      
         

 

### **副本状态转换**

           
    AVAILABLE (在馆可借)      
        ├─ 借出 ──\> ON\_LOAN (外借中)      
        └─ 维修 ──\> MAINTENANCE (维修中)      
           
    ON\_LOAN (外借中)      
        ├─ 归还(正常) ──\> AVAILABLE (在馆可借)      
        ├─ 归还(损坏) ──\> MAINTENANCE (维修中)      
        └─ 归还(丢失) ──\> LOST (丢失)      
           
    MAINTENANCE (维修中)      
        └─ 维修完成 ──\> AVAILABLE (在馆可借)      
           
    LOST (丢失)      
        └─ 找回 ──\> AVAILABLE (在馆可借)      
           
    任意状态 ──\> REMOVED (下架)      
         

 

### **罚款状态转换**

           
    UNPAID (未缴)      
        ├─ 缴费 ──\> PAID (已缴)      
        └─ 减免 ──\> WAIVED (已减免)      
         

 

 

## **七、通知类型说明**

| 通知类型 | 类型标识 | 触发条件 | 通知内容示例 |
| :---- | :---- | :---- | :---- |
| 到期提醒 | DUE\_REMINDER | 借阅到期前N天（N由策略配置） | 您借阅的《XXX》将于3天后到期，请及时归还 |
| 逾期提醒 | OVERDUE\_REMINDER | 逾期N天（N由策略配置） | 您借阅的《XXX》已逾期1天，请尽快归还 |
| 预约到书 | RESERVATION\_READY | 预约晋升为可取书状态 | 您预约的《XXX》已到，请在3天内取书 |
| 预约过期 | RESERVATION\_EXPIRED | 取书截止时间已过 | 您预约的《XXX》已超时未取，预约已失效 |
| 系统公告 | SYSTEM\_ANNOUNCEMENT | 管理员发布 | 系统将于本周六进行维护，请知悉 |

 

## **八、接口规范说明**

### **统一响应格式**

           
    {      
      "code": 200,      
      "message": "操作成功",      
      "data": { ... }      
    }      
         

 

### **错误码说明**

| 错误码 | 说明 |
| :---- | :---- |
| 200 | 操作成功 |
| 400 | 请求参数错误 |
| 401 | 未授权，需要登录 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 409 | 资源冲突（如重复注册） |
| 422 | 业务逻辑错误（如不可借、不可续借） |
| 500 | 服务器内部错误 |

### **分页参数**

所有列表接口统一使用以下分页参数：

* page: 当前页码（从1开始）

* pageSize: 每页数量（默认20，最大100）

 

### **认证方式**

* 会员端和管理端使用 JWT Token 认证

* Token 在请求头中传递：Authorization: Bearer \<token\>

* Token 有效期：7天
